
# מערכת המרה ללשון הקודש - היברידית (כללים + AI)

המלצה: **ארכיטקטורה דו-שכבתית** - שכבת כללים דטרמיניסטית (מהירה, זולה, צפויה) + שכבת AI אופציונלית (חכמה, מטפלת בהקשר). שתיהן ניתנות להפעלה בנפרד או יחד.

## איך עובד התהליך

```text
תמלול גולמי
   ↓
[שכבה 1] מנוע כללים (regex + מילון) — מיידי, חינם
   ↓
[שכבה 2] AI עם פרומפט מומחה — אופציונלי, מטפל בהקשר
   ↓
טקסט סופי + diff להצגה למשתמש
```

## רכיבי המערכת

### 1. בסיס ידע (Knowledge Base) - חד פעמי
**איסוף חוקי ההגייה האשכנזית ממקורות מקצועיים:**
- Wikipedia (אנגלית+עברית): "Ashkenazi Hebrew", "הגייה אשכנזית"
- Jewish Encyclopedia, YIVO Encyclopedia
- מאמרים אקדמיים על פונולוגיה אשכנזית

אבנה **script חד-פעמי** (`scripts/research-ashkenazi.mjs`) שמשתמש ב-**Firecrawl** (כבר זמין כקונקטור) + **Lovable AI** כדי:
1. לסרוק את המקורות
2. לחלץ את כל חוקי ההמרה (חולם→אוֹי, צירה→ֵיי, קמץ→אָ, ת' רפה→ס, וכו')
3. לייצר JSON מובנה של חוקים + דוגמאות
4. לשמור ב-`src/lib/loshonKodeshRulesBase.json`

זה רץ פעם אחת ויוצר את ברירת המחדל המקיפה - המשתמש לא צריך לחכות.

### 2. מנוע הכללים (שכבה 1) - כבר קיים, אורחיב

הקובץ `src/lib/loshonKodesh.ts` כבר מטפל ב-replacements. אוסיף:

- **קטגוריות חוקים** עם הפעלה/כיבוי לכל אחת:
  - חולם (תוירה→תורה)
  - צירה (בייס→בית)
  - קמץ (אָדם)
  - ת' רפה (שאבעס→שבת)
  - הברה סופית (מויישע→משה)
- **חוקי regex מתקדמים** (לא רק מילים שלמות) - לדוגמה `/וי(?=[ה-ת])/g` להמרת תבנית
- **דירוג עדיפות** - חוקים ספציפיים רצים לפני כלליים

### 3. מילוני אוצר מילים תורני - הרחבה

מסך ניהול עם **טאבים** למילונים נפרדים:
- שמות אישים (אבות, אמהות, חז"ל, אדמו"רים)
- מושגי הלכה
- ספרי קודש ומסכתות
- מועדים וזמנים
- ביטויי תפילה
- אוצר ישיבתי/חסידי/ליטאי

כל מילון = רשימת hotwords + replacements ייעודית. ניתן לייבא/לייצא JSON, ויש "פרופילים" (חסידי/ליטאי/ספרדי-אשכנזי).

### 4. שכבת AI (שכבה 2) - חדש

**Edge Function**: `supabase/functions/loshon-kodesh-ai-fix/index.ts`

- משתמש ב-**Lovable AI Gateway** (LOVABLE_API_KEY כבר קיים, אין צורך במפתח חיצוני)
- מודל: `google/gemini-2.5-pro` (איכות מקסימלית לעברית) או `gemini-2.5-flash` (זול ומהיר)
- מקבל: טקסט אחרי שכבה 1 + פרומפט מערכת מפורט
- מחזיר: טקסט מתוקן + רשימת שינויים

**הפרומפט** (ניתן לעריכה במסך):
```
אתה מומחה בעברית תקנית ובהגייה אשכנזית של לשון הקודש.
המשתמש דיבר בהגייה אשכנזית והטקסט תומלל באופן פונטי.
תפקידך: להמיר לכתיב עברי תקני מלא, תוך הבנת ההקשר התורני.

כללים:
1. חולם אשכנזי (oy) → וֹ
2. צירה אשכנזי (ey) → ֵי  
3. ת' רפה (s) → ת
4. קמץ אשכנזי (o) → קמץ רגיל
5. שמות פרטיים: מויישע→משה, יענקל→יעקב...
6. מונחים: דאורייתא, גמרא, פסוק, רש"י...
7. שמור פיסוק טבעי, חלק לפסקאות לפי נושא
8. אל תוסיף תוכן, רק תקן כתיב

החזר JSON: { "fixed": "...", "changes": [{from, to, reason}] }
```

### 5. ממשק המשתמש

במסך `/loshon-kodesh-rules` אוסיף:
- **טאב "מקורות ומחקר"** - כפתור "עדכן חוקים מהאינטרנט" שמריץ את script המחקר
- **טאב "מילונים"** - ניהול קטגוריות אוצר מילים
- **טאב "AI"** - הפעלה/כיבוי, בחירת מודל, עריכת פרומפט, בדיקת עלות
- **טאב "בדיקה"** - input → הצגת תוצאה אחרי שכבה 1, ואחרי שכבה 2, עם diff צבעוני

ב-`Index.tsx` (אחרי תמלול) - אוסיף **כפתור "תקן לעברית תקנית"** ליד הטרנסקריפט שמריץ את שתי השכבות ומציג את ה-diff לפני אישור.

## טכנולוגיות מומלצות

| צורך | טכנולוגיה | למה |
|---|---|---|
| מחקר אוטומטי באינטרנט | **Firecrawl** (כבר מחובר) | מומלץ ע"י Lovable, מחזיר markdown נקי |
| מודל AI | **Lovable AI Gateway** (Gemini 2.5 Pro) | כבר מוגדר, חינם עד גבול, מצוין לעברית |
| Backend | **Supabase Edge Function** | קיים, מאובטח, מסתיר את ה-API key |
| אחסון חוקים | localStorage + cloud sync (`user_preferences.lk_rules`) | חוצה מכשירים |
| עיבוד טקסט | Regex + lookbehind/lookahead לעברית | מובנה ב-JS |
| הצגת diff | `diff` package (כבר קיים בפרויקט?) או `react-diff-viewer` | חוויית UX טובה |
