# תיקון אזור "תמלול מסונכרן" מול "עריכה מסונכרנת"

## הבעיה היום
שני הטורים ב־`SyncMirrorLayout.tsx` עובדים על מקורות שונים: הימני מציג `lines` (תמלול מקור) והשמאלי מציג `text` (טקסט עריכה). הם לא משתנים יחד, היישור נשבר, אין סרגל צף משותף, ואין גרסת בסיס לחזרה.

## המטרה (לפי הבחירות שלך)
1. **שני הטורים תמיד זהים בתוכן** — שינוי מילה בטור העריכה משנה גם את הטור הימני באותו רגע.
2. **גרסת בסיס שמורה** — לפני שמתחילים לערוך, נשמרת "גרסה מקורית" שאפשר לחזור אליה.
3. **סרגל צף מעל שני הטורים** — כותרות, "עריכה מלאה", "יישור מדויק", סגנון הדגשה, פונט/צבע — הכל בסרגל אחד צף, לא כותרת לכל טור.
4. **שמירה אחת = מקומי + ענן יחד** — כפתור שמירה מפעיל את שני המסלולים במקביל עם טוסט אחד.

## מה ייבנה

### 1. סנכרון תוכן דו־כיווני
- הטור הימני יקרא מאותו מקור (`text`) כמו הטור השמאלי, רק במצב read-only עם הדגשת מילה פעילה.
- כל `onTextChange` מהעורך מעדכן מיידית את שניהם (state אחד מקור־האמת).
- ההתאמה בין מילה־לזמן (`displayTimings`) תחושב מחדש על הטקסט המעודכן, כדי שההדגשה הפעילה תמשיך לעבוד גם אחרי עריכה.

### 2. גרסת בסיס + השוואה
- בכניסה לעורך נשמר snapshot `baselineText` (פעם אחת, ב־ref/localStorage לפי `transcriptId`).
- כפתור חדש בסרגל הצף: **"החזר לבסיס"** (אייקון Undo2) — מאפס את `text` ל־baseline אחרי אישור.
- כפתור נוסף: **"השווה לבסיס"** — מפעיל את `compareMode` הקיים מול ה־baseline במקום מול snapshot אחר.

### 3. סרגל צף אחד מעל שני הטורים
- מאחדים את שתי ה־"כותרות לטור" (`עריכה מלאה` של הטור השמאלי + `תמלול מסונכרן` של הטור הימני) לסרגל אחד `sticky top-0` עם רקע מטושטש (`bg-background/80 backdrop-blur`).
- הסרגל יכיל: שם המקטע + מילה פעילה, סגנון הדגשה, פונט/גודל/צבע, יישור מדויק, החזר לבסיס, השווה לבסיס, עריכה מלאה, שמירה.
- הסרגל מוצמד לראש האזור ונשאר גלוי בגלילה.

### 4. שמירה אחת = מקומי + ענן
- כפתור **"שמור"** בסרגל קורא במקביל ל־`onSaveReplace` (מקומי) ול־סנכרון ענן הקיים (`syncCustomToCloud` / `syncGlobalOverridesToCloud` באותו דפוס שכבר השתמשנו).
- טוסט אחד: "נשמר ✓ מקומי + ענן" או "נשמר מקומית, ענן ינוסה שוב ברקע".
- ה־baseline מתעדכן רק בלחיצה מפורשת על "קבע כבסיס חדש" — שמירה רגילה לא דורסת את הבסיס.

## פרטים טכניים
- קובץ עיקרי: `src/components/SyncMirrorLayout.tsx`.
- הטור הימני (כיום `lines` ב־1281) יוחלף כך שייגזר מ־`text` הנוכחי דרך אותה לוגיקת `splitToLines`, ולא מ־state נפרד.
- `baselineText` יישמר ב־`useRef` + `localStorage` עם מפתח `transcript_baseline_<id>`; ב־mount קוראים אם קיים ולא דורסים.
- אם המשתמש מפעיל "יישור מדויק", אותו טקסט מוזרם לשני הצדדים → היישור מילימטרי באופן טבעי, כי המקור זהה.
- כל ההתנהגויות הקיימות (עריכה חופשית, עריכה מלאה, popovers של פונט/צבע, סימון טקסט) נשארות.
